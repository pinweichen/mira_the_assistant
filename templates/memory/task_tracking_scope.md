**Why:** {{USER_NAME}} doesn't want the board cluttered with meta-tasks (e.g., "update CLAUDE.md," "change how scheduling works"). Those are configuration, not work.

**How to apply:** If {{USER_NAME}} asks to change how the assistant operates or add a feature to the workflow, do it but don't log it as a task. Only log tasks that represent actual work they need to get done.
